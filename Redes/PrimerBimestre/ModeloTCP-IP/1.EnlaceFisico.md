# Modelo TCP/IP – Capa de Acceso a la Red (Capa 1)

## Descripción General
La **Capa de Acceso a la Red** (también llamada *Link Layer* o *Network Interface*) es la capa más baja del modelo TCP/IP. Se encarga de cómo los datos se transmiten físicamente a través del medio de comunicación y de cómo los dispositivos acceden a ese medio.

Esta capa no está estrictamente definida en el modelo TCP/IP, ya que depende de las tecnologías de red utilizadas. Por eso, suele considerarse una combinación de funciones que en el modelo OSI están separadas.

---

## Funciones Principales

### 1. Transmisión de Datos
- Convierte los datos en señales eléctricas, ópticas o de radio.
- Define cómo se envían los bits a través del medio físico (cableado o inalámbrico).

### 2. Acceso al Medio
- Determina cómo múltiples dispositivos comparten el mismo canal de comunicación.
- Ejemplo: mecanismos como CSMA/CD (Ethernet tradicional).

### 3. Direccionamiento Físico
- Utiliza direcciones MAC (Media Access Control) para identificar dispositivos dentro de una red local.
- Estas direcciones son únicas para cada interfaz de red.

### 4. Detección de Errores
- Detecta errores en la transmisión de tramas (frames).
- Utiliza técnicas como CRC (Cyclic Redundancy Check).

### 5. Encapsulación de Datos
- Recibe paquetes de la capa de Internet y los encapsula en tramas.
- Agrega información de control (cabeceras y trailers).

---

## Protocolos y Tecnologías Comunes

- Ethernet
- Wi-Fi (IEEE 802.11)
- ARP (Address Resolution Protocol)
- PPP (Point-to-Point Protocol)

---

## Comparación con el Modelo OSI

En el modelo OSI, las funciones de esta capa están divididas en dos capas separadas:

### Capa Física (Physical Layer)
- Maneja la transmisión de bits sin estructura.
- Define características eléctricas, mecánicas y de señalización.

### Capa de Enlace de Datos (Data Link Layer)
- Maneja tramas (frames).
- Controla el acceso al medio.
- Gestiona direcciones MAC y detección de errores.

---

## Diferencias Clave con OSI

- En TCP/IP, ambas capas (Física y Enlace) están combinadas en una sola.
- TCP/IP no define estándares estrictos para esta capa, lo que permite mayor flexibilidad.
- OSI separa claramente responsabilidades, mientras que TCP/IP las agrupa.

---

## Ejemplo de Funcionamiento

1. Un dispositivo quiere enviar datos a otro en la misma red.
2. La capa de Internet entrega un paquete IP a la capa de Acceso a la Red.
3. Se utiliza ARP para obtener la dirección MAC del destino.
4. El paquete se encapsula en una trama Ethernet.
5. La trama se transmite como señal a través del medio físico.

---

## Conclusión

La Capa de Acceso a la Red es fundamental porque conecta el mundo lógico (direcciones IP y paquetes) con el mundo físico (señales y hardware). Aunque en TCP/IP está menos formalizada que en OSI, cumple un rol crítico al garantizar que los datos puedan viajar efectivamente entre dispositivos dentro de una red local.
