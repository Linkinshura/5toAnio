# Modelo TCP/IP – Capa de Internet (Capa 2)

## Descripción General
La **Capa de Internet** es la segunda capa del modelo TCP/IP y se encarga del **direccionamiento lógico** y el **enrutamiento de los paquetes** entre redes. Su objetivo principal es permitir que los datos viajen desde un origen hasta un destino, incluso si están en redes diferentes.

Esta capa introduce el concepto de **paquetes IP** y opera de forma independiente del medio físico utilizado.

---

## Funciones Principales

### 1. Direccionamiento Lógico
- Asigna direcciones IP únicas a cada dispositivo en la red.
- Permite identificar origen y destino de los paquetes.
- Puede trabajar con IPv4 (32 bits) o IPv6 (128 bits).

### 2. Enrutamiento (Routing)
- Determina la mejor ruta para enviar los paquetes desde el origen hasta el destino.
- Los routers utilizan tablas de enrutamiento para decidir por dónde enviar cada paquete.

### 3. Fragmentación y Reensamblado
- Divide paquetes grandes en fragmentos más pequeños si la red lo requiere.
- El destino final es responsable de reensamblar los fragmentos.

### 4. Entrega No Confiable (Best Effort)
- No garantiza la entrega de los paquetes.
- No asegura orden, duplicación ni integridad completa.
- Estas responsabilidades se delegan a la capa de Transporte.

---

## Protocolos Principales

### IP (Internet Protocol)
- Protocolo base de la capa.
- Define el formato de los paquetes y el esquema de direccionamiento.

### ICMP (Internet Control Message Protocol)
- Utilizado para enviar mensajes de control y error.
- Ejemplo: mensajes de "host inaccesible" o pruebas de conectividad (ping).

### IPsec
- Proporciona seguridad a nivel de red mediante cifrado y autenticación.

---

## Comparación con el Modelo OSI

La Capa de Internet del modelo TCP/IP corresponde directamente a:

### Capa de Red (Network Layer) en OSI
- Encargada del direccionamiento lógico.
- Maneja el enrutamiento entre redes.
- Define cómo se envían los paquetes entre nodos.

---

## Diferencias Clave con OSI

- En TCP/IP, la capa de Internet está más centrada en el protocolo IP como estándar dominante.
- El modelo OSI define múltiples protocolos posibles, mientras que TCP/IP prácticamente se basa en IP.
- TCP/IP simplifica la implementación al tener menos abstracción teórica.

---

## Ejemplo de Funcionamiento

1. Una aplicación genera datos que se envían a la capa de Transporte.
2. La capa de Transporte encapsula los datos en segmentos.
3. La Capa de Internet agrega una cabecera IP, creando un paquete.
4. Se determina la dirección IP de destino.
5. El paquete es enviado a través de múltiples routers.
6. Cada router decide el siguiente salto hasta llegar al destino final.

---

## Conclusión

La Capa de Internet es el núcleo del modelo TCP/IP, ya que permite la interconexión de múltiples redes a nivel global. Su diseño basado en el protocolo IP y el modelo de entrega "best effort" hace posible la escalabilidad de Internet, delegando la confiabilidad a capas superiores.
